1955 !> The scatter is implemented as a broadcast of all weights then a local
1956 !> computation on each pe to determine with weights to keep based
1957 !> on gsmap information.
1958 !>
1959 !> The algorithm to determine whether a weight belongs on a pe involves
1960 !> creating a couple local arrays (lsstart and lscount) which are
1961 !> the local values of start and length from the gsmap. These are
1962 !> sorted via a bubble sort and then searched via a binary search
1963 !> to check whether a global index is on the local pe.
1964 !>
1965 !> The local buffer sizes are estimated up front based on ngridcell/npes
1966 !> plus 20% (search for 1.2 below). If the local buffer size fills up, then
1967 !> the buffer is reallocated 50% larger (search for 1.5 below) and the fill
1968 !> continues. The idea is to trade off memory reallocation and copy
1969 !> with memory usage. 1.2 and 1.5 are arbitary, other values may
1970 !> result in better performance.
1971 !>
1972 !> Once all the matrix weights have been read, the sMat is initialized,
1973 !> the values from the buffers are copied in, and everything is deallocated.
1974 !
1975 ! !INTERFACE: -----------------------------------------------------------------
